CT112_DEPLOY_NOTE

Роли и ответственность (обязательно):
- Источник 1: GitHub -> ведет Copilot (я)
- Источник 2: npm -> ведет Copilot (я)
- Источник 3: CT112 -> обновляет пользователь, но ТОЛЬКО по командам Copilot
- Итого: 3 источника, 2 пушу/обновляю я, 1 обновляешь ты по моим шагам

Главное правило этого проекта:
1) Сначала пушим исходники в GitHub.
2) Потом публикуем пакет в npm.
3) Для CT112 НЕ делаем локальную сборку ноды из исходников.
4) Для CT112 даем только команды обновления проекта/пакета на сервере.

Правильный порядок релиза:
A. Локально (разработка):
- изменить код
- commit
- push в GitHub

B. Публикация пакета:
- npm version (или вручную обновить version)
- npm publish --access public
- проверить: npm view node-red-contrib-golc-alice@latest version

C. Обновление на CT112 (рабочая среда):
- зайти в CT112
- выполнить команду установки из npm (актуальной версии)
- перезапустить runtime/процесс

Шаблон команд для CT112 (давать пользователю):
1) Проверить, что в npm вышла нужная версия:
   npm view node-red-contrib-golc-alice@latest version

2) Обновить пакет на CT112:
   npm install node-red-contrib-golc-alice@latest
   (или конкретно: npm install node-red-contrib-golc-alice@0.1.25)

3) Перезапустить процесс (по фактическому способу запуска на CT112):
   pm2 restart <name_or_id>
   или
   docker restart <container_name>
   или
   systemctl restart <service_name>

Запрещено путать:
- GitHub = исходники
- npm = установочный пакет
- CT112 = рабочий сервер, где реально крутится связка с Алисой

Короткая формула для памяти:
GitHub -> npm -> CT112 (команды обновления на сервере)
