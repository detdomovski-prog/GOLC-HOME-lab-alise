Цель проекта

Собрать корректный провайдер Yandex Smart Home на Node-RED, обеспечить стабильный endpoint, и документировать процесс так, чтобы он был воспроизводим и понятен новым участникам.

Краткий план работ

1. Привести репозиторий в читабельный вид: структура папок, шаблоны отчётов.
2. Убедиться, что Node-RED flow (`flows.json`) реализует необходимые endpoint'ы:
   - HEAD /v1.0
   - POST /token
   - GET /v1.0/user/devices
   - POST /v1.0/user/devices/query
   - POST /v1.0/user/devices/action
   - POST /v1.0/user/unlink
3. Устранить внешние барьеры (прокси / cloudflared / nginx) — отключить Basic auth на публичном endpoint'е или настроить корректную авторизацию.
4. Прогнать автоматический тест (`scripts/test-endpoints.ps1`) и собрать сырые ответы.
5. Исправить формат ответов (request_id, action_result, Content-Type), если Яндекс возвращает `UNKNOWN_ERROR`.
6. Подготовить финальный отчёт и рекомендации по деплою.

Критерии успеха

- Все endpoint'ы возвращают HTTP 200 и корректный JSON по спецификации Yandex Smart Home.
- Yandex Smart Home не показывает `UNKNOWN_ERROR` в логах при выполнении команд.
- Репозиторий содержит понятную документацию и шаблоны для отчётов.
